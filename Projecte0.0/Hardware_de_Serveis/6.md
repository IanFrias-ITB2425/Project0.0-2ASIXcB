6\. Seguridad y Firewall (UFW)

Dada la ubicación del servidor en la DMZ (una zona expuesta), es crítico implementar medidas de seguridad de red. Se utilizó UFW (Uncomplicated Firewall) para gestionar las reglas de tráfico mediante una estrategia de Whitelist (bloquear todo por defecto y permitir solo lo esencial).

6\.1. Definición de Políticas por Defecto

Como medida de seguridad base, se bloquearon todas las conexiones entrantes para asegurar que ningún puerto innecesario quedara expuesto accidentalmente.

Bash

sudo ufw default deny incoming

sudo ufw default allow outgoing

6\.2. Apertura de Puertos de Servicio

Se habilitaron explícitamente únicamente los puertos necesarios para la operación del servidor y su administración remota:

SSH (Puerto 22): Necesario para la administración remota mediante el usuario bchecker.

HTTP (Puerto 80): Necesario para que Apache pueda servir la página web a los clientes.

Bash

sudo ufw allow 22/tcp

sudo ufw allow 80/tcp

6\.3. Activación y Verificación

Una vez definidas las reglas, se procedió a habilitar el firewall y verificar su estado para confirmar que las políticas se aplicaron correctamente.

Activación del servicio:

Bash

sudo ufw enable

\# Se confirmó la operación con "y" ante la advertencia de interrupción de SSH.

Verificación de estado: Se utilizó el modo "verbose" para confirmar que la política por defecto es deny y que los puertos 22 y 80 están en estado ALLOW.

Bash

sudo ufw status verbose
